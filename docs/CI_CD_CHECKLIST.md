## Детальная проверка

### GitHub Actions

**Workflow файлы:**
- `.github/workflows/ci.yml` - проверка сборки на push в main/master и PR
- `.github/workflows/deploy-vercel.yml` - проверка сборки (деплой через интеграцию Vercel)

**Ожидаемое поведение:**
- Оба workflow должны запуститься автоматически после push в `main`
- Оба workflow должны завершиться успешно

**Время выполнения:** ~2-5 минут

---

### Vercel

**Конфигурация:** `vercel.json`
- Build: `npm run build`
- Output: `dist`
- Framework: `vite`

**Ожидаемое поведение:**
- Автоматический деплой должен запуститься в течение 1-2 минут после push
- Деплой должен завершиться успешно
- Сайт должен быть доступен

**Время выполнения:** ~2-4 минуты

---

### Railway

**Конфигурация:** `railway.json`
- Build: `npm install && npm run build:server`
- Start: `npm run start:server`

**Ожидаемое поведение:**
- Автоматический деплой должен запуститься после push
- Build должен завершиться успешно
- Сервер должен запуститься и быть доступен

**Время выполнения:** ~3-7 минут

## Troubleshooting

### GitHub Actions не запустились
1. Проверить что workflow файлы в `.github/workflows/`
2. Проверить что ветка называется `main` (не `master`)
3. Проверить права доступа к репозиторию
4. Проверить что коммит действительно отправлен (`git log`)

### Vercel деплой не запустился
1. Проверить Vercel Dashboard → Settings → Git
2. Проверить что интеграция GitHub настроена
3. Проверить что репозиторий подключен правильно
4. Проверить что `vercel.json` находится в корне проекта
5. Проверить логи деплоя в Vercel Dashboard

### Railway деплой не запустился
1. Проверить что проект подключен к GitHub репозиторию
2. Проверить что `railway.json` находится в корне проекта
3. Проверить логи деплоя в Railway Dashboard
4. Проверить что команды сборки правильные (`build:server`, `start:server`)
